## 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请通过以下方式报告：

**📧 邮箱**：security@juezhaotrade.com

**⚠️ 请勿**在公开的 Issue、Pull Request 或任何公开渠道报告安全漏洞。

### 报告内容

请在邮件中包含：

1. **漏洞类型** - 描述漏洞的性质
2. **影响范围** - 说明漏洞可能造成的影响
3. **复现步骤** - 详细的复现步骤
4. **修复建议** - 您认为可行的修复方案（可选）

### 响应时间

- **24 小时内**：确认收到报告
- **7 天内**：提供初步评估
- **30 天内**：发布修复版本（如可行）

## 安全特性

### 密钥加密

觉照验证器使用鸿蒙 HUKS（HarmonyOS Universal KeyStore）进行密钥管理：

- **密钥生成**：在设备 TEE（可信执行环境）中生成，符合《GM/T 0028》和《GM/T 0030》规范
- **加密算法**：AES-256-GCM，提供认证加密
- **密钥存储**：密钥永远不会以明文形式存储或传输

### 数据隔离

- 应用不申请网络权限，无法访问互联网
- 所有数据仅存储在设备本地
- 卸载应用后，所有数据将被清除

### 代码审计

- 核心加密代码遵循最小权限原则
- 欢迎社区进行安全审计

## 已知限制

1. **设备安全性**：应用安全性依赖于设备本身的安全性（如设备被 root 等）
2. **备份风险**：如果提供备份功能，备份文件应妥善保管，避免泄露
3. **屏幕录制**：使用时请注意屏幕录制或截图可能导致密钥泄露

## 更新策略

我们会持续关注并修复安全问题：

- 安全漏洞修复将尽快发布
- 重要安全更新会通过 GitHub Releases 通知
- 建议用户保持使用最新版本

## 第三方依赖

本项目使用的第三方依赖均经过安全评估：

- `@ohos.security.huks` - 鸿蒙官方密钥服务
- `@ohos.data.relationalStore` - 鸿蒙官方数据库
- `@ohos.scanBarcode` - 鸿蒙官方扫码服务
- `@ohos.security.cryptoFramework` - 鸿蒙官方加密框架

以上均为鸿蒙操作系统内置服务，无第三方依赖风险。
