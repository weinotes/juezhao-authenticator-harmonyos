## 安全目标 | Security Objectives

觉照验证器致力于提供安全可靠的离线双因素认证服务：

1. **密钥安全** - 密钥使用硬件级加密存储，永不明文暴露
2. **传输安全** - 应用无网络权限，密钥永不离开设备
3. **存储安全** - 使用设备 TEE/SE 安全芯片保护
4. **代码安全** - 开源可审计，无后门、无恶意代码

## 安全架构 | Security Architecture

### 密钥管理

```
用户输入密钥
    ↓
Base32 解码
    ↓
HUKS AES-256-GCM 加密 ← [TEE/SE 安全芯片]
    ↓
SQLite 数据库存储（密文）
```

### 安全保障措施

| 措施 | 说明 | 符合标准 |
|------|------|----------|
| TEE/SE | 密钥存储在可信执行环境 | GM/T 0028 |
| AES-256-GCM | 军用级加密算法 | FIPS 140-2 |
| 硬件绑定 | 密钥与设备绑定 | - |
| 无网络 | 完全离线，无数据传输 | 华为安全要求 |

## 隐私保护 | Privacy Protection

### 隐私声明

1. **数据收集** - 本应用不收集任何个人信息
2. **数据存储** - 所有数据仅存储在用户设备本地
3. **数据传输** - 应用不申请网络权限，无数据传输
4. **数据删除** - 卸载应用后，所有数据将被清除

### 权限清单

| 权限 | 用途 | 是否必需 |
|------|------|----------|
| 相机权限 | 扫描二维码添加账号 | 是 |
| 网络权限 | 无 | 否（明确不申请） |

### 华为应用市场隐私声明

本应用符合华为应用市场隐私合规要求，详见上架时的隐私政策声明。

## 已知限制 | Known Limitations

1. **设备安全** - 应用安全性依赖于设备本身的安全性
2. **屏幕录制** - 使用时请注意屏幕录制可能导致密钥泄露
3. **备份风险** - 备份文件需妥善保管
4. **系统版本** - 仅支持 HarmonyOS NEXT

## 应急响应 | Incident Response

如发生安全事件，我们将：
1. 立即确认并评估影响范围
2. 24 小时内发布临时修复方案
3. 72 小时内发布正式安全更新
4. 通知受影响用户
5. 提交事件报告至相关机构（如需要）
